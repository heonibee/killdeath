# 킬데스길드 보스 컷/멍 봇 — 설치 안내

리눅스 지식 없이, 전부 웹 브라우저에서 클릭으로 진행합니다.
계정 3개(디스코드 개발자 · 깃허브 · Railway)를 만들고 연결하면 끝입니다.
처음이면 30분~1시간 정도 걸리지만, 한 번 해두면 다음부턴 코드만 고쳐 올리면 됩니다.

---

## 준비물
- 디스코드 계정 (봇을 넣을 서버의 관리자 권한)
- 이 폴더의 파일들: `bot.py`, `bosses.py`, `requirements.txt`, `Procfile`, `runtime.txt`

---

## 1단계 — 디스코드에서 봇 만들기 (토큰 받기)

1. https://discord.com/developers/applications 접속 → 로그인
2. 오른쪽 위 **New Application** → 이름(예: 킬데스보스봇) 입력 → Create
3. 왼쪽 메뉴 **Bot** 클릭 → **Reset Token** → **Yes, do it!** → 나온 토큰을 복사해 **안전한 곳에 저장** (이게 비밀번호입니다. 남에게 노출 금지)
4. 같은 Bot 화면에서 아래로 내려 **Privileged Gateway Intents** 는 전부 꺼도 됩니다(이 봇은 필요 없음).
5. 왼쪽 **Installation** (또는 OAuth2 → URL Generator):
   - **Scopes**: `bot` 과 `applications.commands` 체크
   - **Bot Permissions**: `Send Messages`, `Embed Links`, `Manage Messages`(현황판 고정용), `Read Message History` 체크
   - 아래 생성된 **URL 복사** → 새 탭에서 열기 → 봇을 넣을 **서버 선택** → 승인
   - 이제 서버에 봇이 (오프라인 상태로) 들어와 있습니다.

---

## 2단계 — 깃허브에 코드 올리기

1. https://github.com 가입/로그인
2. 오른쪽 위 **+** → **New repository** → 이름(예: killdes-bossbot) → **Private** 선택 → Create
3. 만들어진 페이지에서 **uploading an existing file** 링크 클릭
4. 이 폴더의 파일들을 끌어다 놓기: `bot.py`, `bosses.py`, `requirements.txt`, `Procfile`, `runtime.txt`
   - ⚠️ `data.json` 이나 토큰이 적힌 파일은 올리지 마세요. (토큰은 3단계에서 따로 넣습니다)
5. **Commit changes** 클릭

---

## 3단계 — Railway에 배포하기

1. https://railway.app 접속 → **Login** → **Login with GitHub** 로 가입
2. **New Project** → **Deploy from GitHub repo** → 방금 만든 저장소 선택
   - 처음이면 Railway가 깃허브 접근 권한을 물어봅니다 → 허용
3. 배포가 시작됩니다. 저장소의 `Procfile`을 읽어 `python bot.py`를 자동 실행해요.
4. **토큰 넣기** (제일 중요):
   - 프로젝트 화면에서 서비스 클릭 → **Variables** 탭 → **New Variable**
   - 이름: `DISCORD_TOKEN`
   - 값: 1단계에서 복사한 **봇 토큰** 붙여넣기 → 저장
5. 저장하면 자동으로 다시 배포됩니다. **Deploy Logs** 에서 `로그인: ...` 이 보이면 성공.

---

## 4단계 — 디스코드에서 사용하기

봇이 온라인이 되면, 봇을 쓸 채널에서:

- **`/현황판`** — 그 채널에 실시간 현황판을 설치하고 상단에 고정합니다. (한 번만)
- 현황판의 **드롭다운 메뉴**에서 보스를 고르면 **🟢컷 / 🔴멍 / ↩취소** 버튼이 뜹니다. 누르면 시각과 누른 사람이 기록되고 현황판이 자동 갱신돼요.
- 빠르게 치고 싶으면 명령어도 됩니다: **`/컷 스펙터`**, **`/멍 나가`** (이름 자동완성 지원)
- **`/로그`** — 최근 컷/멍 기록 보기 (특정 보스만도 가능)
- **`/리셋`** — 현재 타임 수동 초기화

현황판은 **00·03·09·12·21시**에 자동으로 새 타임으로 리셋됩니다(컷/멍 초기화, 로그는 계속 보존).

## 자주 묻는 것

**Q. 내 PC를 꺼도 되나요?**
네. Railway(클라우드)가 24시간 돌리므로 내 컴퓨터·인터넷과 무관합니다.

**Q. 명령어가 안 보여요.**
봇 초대 시 `applications.commands` 스코프가 빠졌을 수 있어요. 1단계 URL을 다시 만들어 재초대하세요. 슬래시 명령은 등록에 최대 1시간 걸릴 수 있습니다.

**Q. 보스 목록이나 시간을 바꾸려면?**
`bosses.py` 파일만 고쳐서 깃허브에 다시 올리면 Railway가 자동 재배포합니다.

**Q. 기록(로그)은 어디 저장되나요?**
Railway 서버의 `data.json` 파일에 쌓입니다. 재배포 시 초기화될 수 있으니, 장기 보관이 중요해지면 알려주세요 — DB(무료 PostgreSQL 등)로 옮기는 방법을 안내하겠습니다.

**Q. 알림(정시 젠 자동 공지)도 넣고 싶어요.**
지금은 빼뒀습니다. 원하면 특정 채널에 "곧 젠" 자동 공지 + 역할 멘션을 추가할 수 있어요.
